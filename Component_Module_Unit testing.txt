   Модульное тестирование(юнит-тустирование) используется для тестирования какого-либо одного логически выделенного
изолированного элемента системы в коде. Это тестирование методом белого ящика и проводится самими разработчиками.
Целью тестирования модуля является не демонстрация правильного функционирования модуля, а демонстрация наличия ошибки в модуле.
   Компонентное тестирование - тип тестирования ПО, при котором тестирование выполняется для каждого отдельного компонента отдельно,
   без интеграции с другими компонентами. Его так же называют модульным тестированием, если смотреть с точки зрения архитектуры.
   Как правило, любое ПО в целом состоит из нескольких компонентов. Тестирование на уровне компонентов имеет дело с тестированием 
   этих компонентов индивидуально. Это один из самых частых типов тестирования черного ящика, который проводится командой QA.
