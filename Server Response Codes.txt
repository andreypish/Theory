 Коды ответа сервера - это трехзначное число, состоящее из 3-х арабских цифр. За кодом ответа
следует поясняющая фраза, которая объясняет причину ответа. Первая цифра кода поясняет класс ответа.

  КЛАССЫ КОДОВ ОТВЕТА СЕРВЕРА:
  1хх - Информационный - означает, что сервер принял запрос и обрабатывает его дальше.
  2хх - Успешный - означает, что сервер успешно обработал запрос.
  3хх - Перенаправление - сервер сообщает, что для выполнения запроса необходимы доп. действия(например, переход на другой URL)
  4хх - Ошибка пользователя - сервер не может успешно выполнить запрос по причине ошибки на стороне клиента.
  5хх - Ошибка сервера - сервер не может успешно выполнить запрос по причине ошибки на стороне сервера. 
  
  1 КЛАСС - INFORMATIONL
 100 Continue(Продолжить)
Это промежуточный ответ сервера, информирующий, что сервер принял запрос и будет дальше совершать обработку.
 101 Switching Protocols(Переключение протоколов)
Этот код присылается в ответ на запрос клиента, содержащий заголовок Upgrade. 
 102 Processing(В обработке)
Сервер получил запрос, обрабатывает его, но обработка еще не завершена и ьребует дополнительного времени.
Клиент не должен разрывать соединения, а обнулить таймер и дождаться следующего ответа. 
Данный код используется в протоколе WebDAV(надстройка HTTP для работы с файлами).
 103 Early Hints(Ранние подсказки)
Прежде всего, данный код используется с заголовком Link. Используется чтобы клиент мог получить предварительные
для загрузки ресурсы, пока сервер готовит основной ответ.

  2 КЛАСС - SUCCESS
 200 OK(Успешно)
Означает успешную обработку запроса.
 201 Created(Создано)
Означает успешное создание нового ресурса, в результате запроса клиента. 

  3 КЛАСС - REDIRECTION
 301 Moved Permanently(Перемещение на постоянной основе)
Указынный URL был перемещен или не существует и пользователю предлогается переход на новый URL. Адрес перемещен навсегда.
 302 Found(Найдено)
Указанный URL был временно перенесен на другой адресс. 
 304 Not Modified(Не изменено)
Данный код используется при кэшировании и означает, что ресурс не был изменен. Клиент может использовать версию из кэша.
Нагрузка на сервер не увеличивается.

  4 КЛАСС CLIENT ERROR
 401 Unauthorized(Неавторизован)
Для получения запрашиваемого ответа нужна аутентификация.
 403 Forbidden(Доступ запрещен)
Запрос получен, но сервер не может его выполнить по причине ограничений доступа. 
 404 Not Found(Не найдено)
Сервер получил запрос, но не нашел документа по запрашиваемому URL.
 410 Gone(удален)
Сервер сообщает, что документ ранее находился на запрашиваемом адресе, но сейчас удален навсегда. 
 451 Unvailable For Legal Reasons (Недоступно по юридическим причинам)
Означает, что доступ к серверу закрыт из-за его запрета на государственном уровне или по решению суда.
 
  5 КЛАСС SERVER ERROR
 500 Internal Server Error(Внутренняя ошибка сервера)
Сервер не может выполнить запрос из-за некой внутренней ошибки.
 503 Service Unavailable (Сервис недоступен)
Сервер не может выполнить запрос по техническим причинам.
 504 Gateway Timeout (Время ожидания шлюза истекло)
Сервер, который выполняет роль шлюза не дождался ответа от вышестоящего сервера.
